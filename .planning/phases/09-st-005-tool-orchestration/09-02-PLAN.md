    Face 4 (WHERE): Real-world examples using MCP tool stacks, n8n workflow automation, microservice dependencies. 3-4 examples with tool names, dependency paths, and intervention recommendations.

    Face 5 (WHEN): Decision triggers -- after adding a new tool to stack, after a tool failure cascades, during architecture review, when health scores cross into At Risk or Critical. Exclusions: single standalone tools, tools with no dependencies.

    Face 6 (APPLY): Practice exercise directing to Tool Orchestration Analyzer artifact. MCP tool integration example with analyze_tool_orchestration. Reference the course's own tool stack as practice material.
