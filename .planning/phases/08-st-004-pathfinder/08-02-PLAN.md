    Face 1 -- WHAT: Define agent feedback loops. An agent feedback loop is a circular causal structure within an AI agent architecture where an agent's output feeds back through tools, evaluators, memory, or constraints to influence the agent's subsequent behavior. Cover reinforcing loops (retry storms, cost spirals, capability snowballs) and balancing loops (rate limiters, budget caps, timeout constraints). Boundaries: not simple error handling (that's linear), not multi-agent communication (out of scope), not chatbot conversation flow (no agency). Graph relationships: Parent = Feedback Loop Dynamics, Prerequisites = ST-001 (Reinforcing Feedback Loops), Enables = ST-005 (Tool Orchestration). Target: 9/10.

    Face 2 -- WHY: Business problem is that agent architectures contain hidden feedback loops that cause runaway costs, cascading failures, and retry storms that teams diagnose as "bugs" instead of structural dynamics. Use the retry storm scenario from CONTEXT.md as the cost-of-ignorance example: agent calls tool, fails, retries, hits rate limit, timeouts increase, triggering more retries. Persona stakes for CxO (agent cost overrun), mid-market (production agent stability), solopreneur (agent reliability on budget). Target: 9/10.

    Face 3 -- HOW: Core mechanics. Explain the 5 agent component types (agent=hexagon, tool=rectangle, memory=cylinder, evaluator=diamond, constraint=octagon) from agent-visual-vocabulary.md. Step-by-step: (1) Identify agent components, (2) Map causal links with polarity, (3) Trace cycles using DFS, (4) Classify loops (even negatives = reinforcing, odd = balancing), (5) Score severity via loop gain, (6) Target highest-gain link for intervention. Technical connection: this applies ST-001 loop detection to agent domain. Target: 8/10.

    Face 4 -- WHERE: Real-world examples. Retry storm loop (agent > tool > error handler > agent), cost spiral loop (agent complexity > token usage > spend > scaling pressure > more agents), capability snowball (better tools > more capabilities > more users > more data > better tools). Each example names the loop type and identifies the highest-leverage intervention point. Target: 8/10.

    Face 5 -- WHEN: Decision framework. Use this when: designing a new agent, debugging agent failures, cost optimization, scaling agent architectures. Trigger conditions: "my agent keeps retrying," "costs are growing faster than usage," "adding tools makes things worse." Do NOT use when: simple chatbot (no loops), one-shot API calls (no feedback), manual workflows (no automation). Target: 8/10.

    Face 6 -- APPLY: Practice exercise. Direct student to the interactive artifact: "Open the Agent Feedback Loop Builder artifact. Load the retry storm example. Study the pre-annotated loops. Then switch to practice mode and map YOUR agent architecture (or use the 5-node template). Predict each loop's type and behavior. Reveal the analysis and compare your predictions." Reference the MCP tool: "Call analyze_agent_feedback_loops with your agent components and links to verify the artifact's analysis matches the programmatic result." Target: 8/10.
